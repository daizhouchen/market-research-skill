# 市场研究分析框架

## 分析流水线

```
原始数据 → 清洗 & 标准化 → 模式识别 → 交叉验证 → 因果推断 → 机会发现 → 行动建议
```

---

## 第一步：数据清洗

### 对齐与标准化
- **时间范围对齐**：确保所有数据源覆盖相同时间段（默认12个月）
- **地理范围对齐**：统一地区筛选条件（如仅美国市场、全球市场等）
- **格式标准化**：统一货币（USD）、评分体系（归一化至0-1）、日期格式（ISO 8601）

### 噪声去除
- **Reddit**：过滤广告帖、自我推广帖（检测 URL 密度、账号年龄、karma 值）
- **Amazon**：识别并排除虚假评论（verified purchase 过滤、评论文本相似度检测）
- **Google Trends**：排除因新闻事件导致的异常峰值（标注但不删除）
- **通用规则**：数据量不足10条的维度标记为"低置信度"

## 第二步：模式识别

| 观察到的模式 | 分析方法 | 含义解读 |
|---|---|---|
| 搜索趋势上升 + 斜率加速 | Google Trends 线性回归 + 二阶导数 | 加速增长期，进入窗口期 |
| 搜索趋势上升 + 斜率趋平 | Google Trends 线性回归 + 斜率变化 | 增长放缓，接近饱和 |
| 周期性波动 | 变异系数(CV)分析，CV > 0.3 | 存在季节性，需对应时间节点规划 |
| 高评分 + 少评论数 | Amazon/App Store 评分-评论交叉 | 小众市场，用户满意但市场未被充分开发 |
| 低评分 + 多评论数 | Amazon/App Store 评分-评论交叉 | 需求旺盛但现有产品质量差，改进空间大 |
| 差评集中在某一功能 | 评论文本聚类分析 | 明确的产品改进方向 |
| "wish/hope/want" 高频出现 | Reddit/评论 NLP 情感分析 | 未被满足的需求，潜在产品方向 |
| 价格空白带 + 高搜索量 | Amazon 价格分布 + Google Trends | 定价机会，该价位段供给不足 |
| 多家新公司获得融资 | Crunchbase 融资数据分析 | 资本看好但竞争正在加剧 |

## 第三步：交叉验证规则

### 核心原则
> **任何出现在"洞察"或"建议"中的结论，必须有至少2个独立数据源支撑。**

### 交叉验证矩阵

| 结论类型 | 必需证据组合（至少满足一行） |
|---|---|
| "市场正在增长" | Google Trends 上升趋势 + (新融资事件 OR 新产品上架增加 OR Reddit 讨论量增长) |
| "存在未满足需求" | Reddit/评论中用户抱怨 + (搜索趋势上升 OR 现有产品低评分 OR "wish/hope/want"关键词) |
| "市场竞争激烈" | 多家融资企业 + (Amazon 产品数量多 OR 价格战迹象 OR SimilarWeb 多站流量高) |
| "产品方向X有机会" | 用户需求频次高 + (现有产品该功能缺失 OR 该功能差评集中 OR 相关搜索趋势上升) |
| "价格定位在X区间" | 价格带分析空白 + (用户价格敏感度数据 OR 竞品定价对比 OR 搜索量与价格交叉) |
| "季节性明显" | Google Trends 周期模式 + (Amazon 销量波动 OR Reddit 讨论周期性) |
| "目标用户是X群体" | Reddit 用户画像 + (评论用户特征 OR 搜索地域分布 OR 竞品定位分析) |

### 验证失败处理
- 仅有单一数据源支撑的结论 → 标记为"待验证假设"，不放入主要结论
- 数据源之间相互矛盾 → 明确标注矛盾点，列出两方证据，不强行得出结论

## 第四步：因果推断规则

### 表述规范
- **禁止**："因为A所以B"、"A导致了B"
- **允许**："A和B共同出现/相关联"、"A与B存在正相关"、"在A发生的同时，B也在增长"

### 置信度等级

| 等级 | 条件 | 标记 |
|---|---|---|
| 🟢 高置信度 | 多数据源支撑 + 逻辑链完整 | `[置信度: 高]` |
| 🟡 中置信度 | 两个数据源 或 一个强数据源 + 逻辑合理 | `[置信度: 中]` |
| 🔴 低置信度 | 单一数据源 或 数据源间存在矛盾 | `[置信度: 低]` |

### 事实 vs 推断 标注
- **事实(Fact)**：直接从数据中读取的数值或状态（如"该关键词12个月搜索量增长了150%"）
- **推断(Inference)**：基于数据推导出的判断（如"该市场可能正在进入快速增长期"）
- 输出时必须明确标注 `[事实]` 或 `[推断]`

## 第五步：机会发现框架

### 机会评分公式

```
机会得分 = 需求强度 × 供给缺口 × 可行性
```

| 维度 | 评分要素 | 权重 |
|---|---|---|
| **需求强度** | 搜索趋势增长率、用户讨论热度、评论中需求频率、融资热度 | 40% |
| **供给缺口** | 现有产品评分、功能覆盖率、价格带空白、竞争者数量 | 35% |
| **可行性** | 技术门槛、资金门槛、监管壁垒、时间窗口 | 25% |

### 评分标准（各维度1-5分）
- **5分**：极强信号（多数据源一致性高）
- **4分**：强信号（主要数据源支持）
- **3分**：中等信号（数据支持但有不确定性）
- **2分**：弱信号（有限数据支持）
- **1分**：极弱信号（几乎无数据支持或信号矛盾）

## 五大关键问题框架（最终综合）

### Q1: 这个市场值得进入吗？
- 输出：市场吸引力评级（1-10分）+ 理由
- 考量因素：市场规模、增长速度、竞争格局、进入壁垒、利润空间

### Q2: 最大的机会在哪里？
- 输出：Top 2-3 个具体产品/服务方向
- 每个方向需包含：目标用户、核心差异化、预估市场规模

### Q3: 主要风险是什么？
- 输出：按严重程度排序的风险列表
- 分类：市场风险、竞争风险、技术风险、监管风险、执行风险

### Q4: 如果进入，差异化方向是什么？
- 输出：推荐的差异化策略
- 考量：功能差异化、价格差异化、用户体验差异化、渠道差异化

### Q5: 下一步行动是什么？
- 输出：3个具体的行动项，按优先级排序
- 每项包含：具体任务、预期产出、建议时间框架

## 各维度专项分析方法

### 趋势分析（Trend Analysis）

- **数据范围**：12个月周级数据
- **核心指标**：
  - 线性回归斜率（判断整体趋势方向和强度）
  - 季节性检测（变异系数 CV > 0.3 判定为存在季节性）
  - 机会关键词挖掘（增长率 > 100% 的相关搜索词）
  - 地域差异分析（各地区搜索热度对比）
- **输出格式**：趋势概览 + 季节性模式 + 机会关键词列表 + 地域热力图

### 产品竞争分析（Product Competition）

- **数据采集**：Amazon/App Store/Google Play 各品类 TOP 20 产品
- **分析维度**：
  - 功能矩阵构建（产品 × 功能 的二维表）
  - 价格带分析（价格分布直方图，识别空白带）
  - 差异化分析（各产品独有功能、用户提及的独特卖点）
  - 评分与评论趋势（近期评分变化，识别上升/下滑产品）
- **输出格式**：竞争格局地图 + 功能对比矩阵 + 价格带图 + 差异化机会

### 用户需求分析（User Demand）

- **数据来源**：Reddit 帖子/评论、Amazon/App 评论
- **分析流程**：
  1. 提取用户需求和痛点关键句
  2. 分类：功能需求 / 性能需求 / 体验需求 / 价格需求
  3. 按 频率 × 强度 排序
  4. 用户群体细分（按使用场景、专业程度、价格敏感度）
  5. "wish/hope/want" 模式挖掘未满足需求
- **输出格式**：需求优先级矩阵 + 用户画像 + 未满足需求清单

### 竞争公司分析（Competitor Companies）

- **识别方式**：Crunchbase 搜索 + ProductHunt 新产品 + 已知竞品
- **分析维度**：
  - 融资情况：总融资额、轮次分布、最近融资时间
  - 竞争强度指数：参与者数量 × 平均融资额 × 新进入者比例
  - 近6个月新进入者分析（关注新玩家的差异化方向）
  - 头部玩家 vs 长尾玩家的市场份额估算
- **输出格式**：竞争格局概览 + 融资时间线 + 竞争强度评级 + 新进入者列表
