# 市场研究分析框架

## 分析流水线

```
原始数据 → 清洗 & 标准化 → 模式识别 → 交叉验证 → 因果推断 → 机会发现 → 行动建议
```

---

## 第一步：数据清洗

### 对齐与标准化
- **时间范围对齐**：确保所有数据源覆盖相同时间段（默认12个月）
- **地理范围对齐**：统一地区筛选条件（如仅美国市场、全球市场等）
- **格式标准化**：统一货币（USD）、评分体系（归一化至0-1）、日期格式（ISO 8601）

### 噪声去除
- **Reddit**：过滤广告帖、自我推广帖（检测 URL 密度、账号年龄、karma 值）
- **Amazon**：识别并排除虚假评论（verified purchase 过滤、评论文本相似度检测）
- **Google Trends**：排除因新闻事件导致的异常峰值（标注但不删除）
- **通用规则**：数据量不足10条的维度标记为"低置信度"

## 第二步：模式识别

| 观察到的模式 | 分析方法 | 含义解读 |
|---|---|---|
| 搜索趋势上升 + 斜率加速 | Google Trends 线性回归 + 二阶导数 | 加速增长期，进入窗口期 |
| 搜索趋势上升 + 斜率趋平 | Google Trends 线性回归 + 斜率变化 | 增长放缓，接近饱和 |
| 周期性波动 | 变异系数(CV)分析，CV > 0.3 | 存在季节性，需对应时间节点规划 |
| 高评分 + 少评论数 | Amazon/App Store 评分-评论交叉 | 小众市场，用户满意但市场未被充分开发 |
| 低评分 + 多评论数 | Amazon/App Store 评分-评论交叉 | 需求旺盛但现有产品质量差，改进空间大 |
| 差评集中在某一功能 | 评论文本聚类分析 | 明确的产品改进方向 |
| "wish/hope/want" 高频出现 | Reddit/评论 NLP 情感分析 | 未被满足的需求，潜在产品方向 |
| 价格空白带 + 高搜索量 | Amazon 价格分布 + Google Trends | 定价机会，该价位段供给不足 |
| 多家新公司获得融资 | Crunchbase 融资数据分析 | 资本看好但竞争正在加剧 |

## 第三步：交叉验证规则

### 核心原则
> **任何出现在"洞察"或"建议"中的结论，必须有至少2个独立数据源支撑。**

### 交叉验证矩阵

| 结论类型 | 必需证据组合（至少满足一行） |
|---|---|
| "市场正在增长" | Google Trends 上升趋势 + (新融资事件 OR 新产品上架增加 OR Reddit 讨论量增长) |
| "存在未满足需求" | Reddit/评论中用户抱怨 + (搜索趋势上升 OR 现有产品低评分 OR "wish/hope/want"关键词) |
| "市场竞争激烈" | 多家融资企业 + (Amazon 产品数量多 OR 价格战迹象 OR SimilarWeb 多站流量高) |
| "产品方向X有机会" | 用户需求频次高 + (现有产品该功能缺失 OR 该功能差评集中 OR 相关搜索趋势上升) |
| "价格定位在X区间" | 价格带分析空白 + (用户价格敏感度数据 OR 竞品定价对比 OR 搜索量与价格交叉) |
| "季节性明显" | Google Trends 周期模式 + (Amazon 销量波动 OR Reddit 讨论周期性) |
| "目标用户是X群体" | Reddit 用户画像 + (评论用户特征 OR 搜索地域分布 OR 竞品定位分析) |

### 验证失败处理
- 仅有单一数据源支撑的结论 → 标记为"待验证假设"，不放入主要结论
- 数据源之间相互矛盾 → 明确标注矛盾点，列出两方证据，不强行得出结论

## 第四步：因果推断规则

### 表述规范
- **禁止**："因为A所以B"、"A导致了B"
- **允许**："A和B共同出现/相关联"、"A与B存在正相关"、"在A发生的同时，B也在增长"

### 置信度等级

| 等级 | 条件 | 标记 |
|---|---|---|
| 🟢 高置信度 | 多数据源支撑 + 逻辑链完整 | `[置信度: 高]` |
| 🟡 中置信度 | 两个数据源 或 一个强数据源 + 逻辑合理 | `[置信度: 中]` |
| 🔴 低置信度 | 单一数据源 或 数据源间存在矛盾 | `[置信度: 低]` |

### 事实 vs 推断 标注
- **事实(Fact)**：直接从数据中读取的数值或状态（如"该关键词12个月搜索量增长了150%"）
- **推断(Inference)**：基于数据推导出的判断（如"该市场可能正在进入快速增长期"）
- 输出时必须明确标注 `[事实]` 或 `[推断]`

## 第五步：机会发现框架

### 机会评分公式

```
机会得分 = 需求强度 × 供给缺口 × 可行性
```

| 维度 | 评分要素 | 权重 |
|---|---|---|
| **需求强度** | 搜索趋势增长率、用户讨论热度、评论中需求频率、融资热度 | 40% |
| **供给缺口** | 现有产品评分、功能覆盖率、价格带空白、竞争者数量 | 35% |
| **可行性** | 技术门槛、资金门槛、监管壁垒、时间窗口 | 25% |

### 评分标准（各维度1-5分）
- **5分**：极强信号（多数据源一致性高）
- **4分**：强信号（主要数据源支持）
- **3分**：中等信号（数据支持但有不确定性）
- **2分**：弱信号（有限数据支持）
- **1分**：极弱信号（几乎无数据支持或信号矛盾）

## 五大关键问题框架（最终综合）

### Q1: 这个市场值得进入吗？
- 输出：市场吸引力评级（1-10分）+ 理由
- 考量因素：市场规模、增长速度、竞争格局、进入壁垒、利润空间

### Q2: 最大的机会在哪里？
- 输出：Top 2-3 个具体产品/服务方向
- 每个方向需包含：目标用户、核心差异化、预估市场规模

### Q3: 主要风险是什么？
- 输出：按严重程度排序的风险列表
- 分类：市场风险、竞争风险、技术风险、监管风险、执行风险

### Q4: 如果进入，差异化方向是什么？
- 输出：推荐的差异化策略
- 考量：功能差异化、价格差异化、用户体验差异化、渠道差异化

### Q5: 下一步行动是什么？
- 输出：3个具体的行动项，按优先级排序
- 每项包含：具体任务、预期产出、建议时间框架

## 各维度专项分析方法

### 趋势分析（Trend Analysis）

- **数据范围**：12个月周级数据
- **核心指标**：
  - 线性回归斜率（判断整体趋势方向和强度）
  - 季节性检测（变异系数 CV > 0.3 判定为存在季节性）
  - 机会关键词挖掘（增长率 > 100% 的相关搜索词）
  - 地域差异分析（各地区搜索热度对比）
- **输出格式**：趋势概览 + 季节性模式 + 机会关键词列表 + 地域热力图

### 产品竞争分析（Product Competition）

- **数据采集**：Amazon/App Store/Google Play 各品类 TOP 20 产品
- **分析维度**：
  - 功能矩阵构建（产品 × 功能 的二维表）
  - 价格带分析（价格分布直方图，识别空白带）
  - 差异化分析（各产品独有功能、用户提及的独特卖点）
  - 评分与评论趋势（近期评分变化，识别上升/下滑产品）
- **输出格式**：竞争格局地图 + 功能对比矩阵 + 价格带图 + 差异化机会

### 用户需求分析（User Demand）

- **数据来源**：Reddit 帖子/评论、Amazon/App 评论
- **分析流程**：
  1. 提取用户需求和痛点关键句
  2. 分类：功能需求 / 性能需求 / 体验需求 / 价格需求
  3. 按 频率 × 强度 排序
  4. 用户群体细分（按使用场景、专业程度、价格敏感度）
  5. "wish/hope/want" 模式挖掘未满足需求
- **输出格式**：需求优先级矩阵 + 用户画像 + 未满足需求清单

### 竞争公司分析（Competitor Companies）

- **识别方式**：Crunchbase 搜索 + ProductHunt 新产品 + 已知竞品
- **分析维度**：
  - 融资情况：总融资额、轮次分布、最近融资时间
  - 竞争强度指数：参与者数量 × 平均融资额 × 新进入者比例
  - 近6个月新进入者分析（关注新玩家的差异化方向）
  - 头部玩家 vs 长尾玩家的市场份额估算
- **输出格式**：竞争格局概览 + 融资时间线 + 竞争强度评级 + 新进入者列表

## 深度分析方法论（v2 增强）

### 一、市场规模估算方法（TAM/SAM/SOM）

#### 自上而下法（Top-Down）
- 从行业报告中获取全球/区域市场总值
- 按目标细分、地区、价格段层层缩减
- 公式：TAM × 目标细分占比 × 地区占比 = SAM；SAM × 可触达率 = SOM

#### 自下而上法（Bottom-Up）
- 从搜索量推算潜在用户基数
  - Google Trends 搜索指数 → 通过 Keyword Planner 基准转换为绝对搜索量
  - 月搜索量 × 点击率(~30%) × 转化率(行业平均) × 客单价 = 可估算市场
- 从竞品数据反推
  - App 下载量 × ARPU = 单产品收入
  - 已知竞品收入 × 市场份额倒推 = 总市场

#### 三角验证
至少用两种方法估算，取范围而非单一数字。标注数据来源和假设。

#### 生命周期定位
根据以下指标综合判断市场所处阶段：

| 阶段 | 搜索趋势 | 产品数量 | 融资特征 | 价格趋势 |
|------|---------|---------|---------|---------|
| 萌芽期 | 低基数高增长 | <10 | 种子/天使轮为主 | 高，无标准 |
| 成长期 | 高增长 | 快速增加 | A/B轮活跃 | 开始分层 |
| 成熟期 | 增速放缓 | 大量同质化 | C轮+/IPO | 价格战 |
| 衰退期 | 下降 | 开始减少 | 并购为主 | 底部竞争 |

### 二、Porter 五力分析框架

对每一力进行 1-5 分评估：

1. **现有竞争者的竞争强度**
   - 来源数据：竞品数量、融资总额、价格差异度、功能同质化程度
   - 评分标准：竞品>20且功能相似→5分；竞品<5→1分

2. **新进入者威胁**
   - 来源数据：近6月新公司、技术壁垒高低、资金壁垒、品牌壁垒
   - 评分标准：技术壁垒低+资金需求小+新玩家多→5分

3. **替代品威胁**
   - 来源数据：Reddit中提到的替代方案、用户"不解决"的比例
   - 评分标准：替代方案多且迁移成本低→5分

4. **买方议价能力**
   - 来源数据：价格敏感度、评论中价格抱怨频率、免费替代品数量
   - 评分标准：免费替代品多+价格抱怨频繁→5分

5. **供方议价能力**
   - 来源数据：关键技术/平台依赖、供应链集中度
   - 评分标准：高度依赖单一平台(如App Store)→5分

输出：五力雷达图数据 + 综合竞争强度评级

### 三、JTBD（Jobs-to-be-Done）分析框架

从用户评论和讨论中提取用户"任务"，而非表面功能需求。

#### 任务提取模板
```
当我 [情境]，
我想要 [动机/目标]，
这样我就能 [期望结果]。
```

#### 分析步骤
1. 从 Reddit 帖子/评论中提取用户描述使用场景的文本
2. 按上述模板结构化提取 JTBD
3. 区分三类任务：
   - 功能任务（完成某个具体事情）
   - 情感任务（感觉如何）
   - 社交任务（被他人如何看待）
4. 对每个任务评估：
   - 重要性（用户提及频率×情绪强度）
   - 满足度（现有产品解决得好不好）
5. 机会分数 = 重要性 - 满足度
   - 机会分数 > 3 → 高价值未满足需求
   - 机会分数 < 0 → 过度服务（可能存在简化/降价空间）

#### 输出
JTBD 机会图：横轴=满足度，纵轴=重要性，右上角=桌面赌注，左上角=黄金机会

### 四、技术采纳生命周期定位

根据 Geoffrey Moore 的"跨越鸿沟"理论，判断产品/市场处于技术采纳曲线的哪个阶段：

| 阶段 | 用户特征 | 数据信号 |
|------|---------|---------|
| 创新者(2.5%) | 技术极客、尝鲜 | ProductHunt 活跃、GitHub Star 增长、Reddit r/technology 讨论 |
| 早期采用者(13.5%) | 有远见、愿付费 | 融资B轮+、付费用户增长、正面评论开始增多 |
| 鸿沟 | — | 增长停滞、用户从极客到主流的过渡困难 |
| 早期多数(34%) | 实用主义、需要完整方案 | 大量评论、主流媒体报道、大公司入场 |
| 晚期多数(34%) | 保守、价格敏感 | 价格下降、同质化严重、评分趋于平均 |
| 落后者(16%) | 抗拒变化 | 搜索趋势下降、公司倒闭/被收购 |

定位方法：综合搜索趋势增速、产品成熟度、用户评论专业度、媒体报道密度判断。

### 五、高级文本挖掘方法

#### 1. 主题聚类（Topic Clustering）
- 从所有评论/帖子中提取名词短语
- 按共现频率聚类为主题
- 每个主题标注：情感倾向、提及频率、趋势（上升/下降）
- 发现"新兴主题"：近3个月提及频率显著高于前9个月的主题

#### 2. 特征共现分析
- 构建特征共现矩阵：哪些功能经常被一起提及
- 识别"必备组合"（共现率>70%的功能对→用户认为这些应该一起提供）
- 识别"差异化组合"（共现率低但满意度高→可能是独特卖点）

#### 3. 评论速度分析（Review Velocity）
- 追踪评论/帖子数量的时间变化率
- 评论加速 → 市场关注度上升的领先指标
- 按产品分别追踪 → 识别增长最快和下滑最快的竞品

#### 4. 用户迁移路径分析
- 从"我从X换到Y"类型的评论中提取迁移路径
- 构建产品间迁移网络图
- 净迁入为正的产品 = 正在获胜；净迁出的产品 = 正在失去用户
- 迁移原因聚类 → 揭示用户切换的核心驱动力

#### 5. 价格敏感度分析
- 从评论中提取价格相关表述的情感
- 按用户分群分析付费意愿差异
- 识别"价格锚点"：用户反复引用的参考价格
- 弹性估算：价格变化与评论量/评分的相关性

### 六、蓝海画布（Strategy Canvas）

#### 构建方法
1. 从竞品矩阵中提取 6-10 个关键竞争要素
2. 对每个竞品，按各要素评分（1-5）
3. 绘制策略画布：横轴=竞争要素，纵轴=投入/表现水平
4. 寻找蓝海：
   - **消除**：行业习以为常但用户不在意的要素
   - **减少**：被过度竞争的要素（如都在堆砌功能）
   - **提升**：用户真正在意但被忽视的要素
   - **创造**：行业从未提供的全新要素

#### 数据来源
- 竞争要素提取：产品描述 + 用户评论高频词
- 各产品评分：功能覆盖度 + 评论中的满意度评分
- 用户真实关注度：评论提及频率 × 情绪强度

### 七、综合深度分析输出框架

在完成上述所有分析后，按以下结构输出深度洞察：

```
1. 市场定位诊断
   ├─ 市场规模（TAM/SAM/SOM 范围估算）
   ├─ 生命周期阶段
   └─ 技术采纳曲线位置

2. 竞争力学分析
   ├─ Porter 五力评分与雷达图
   ├─ 蓝海画布与差异化空间
   └─ 竞品迁移网络

3. 需求深度图谱
   ├─ JTBD 机会图
   ├─ 主题聚类与新兴趋势
   ├─ 特征共现网络
   └─ 价格敏感度分层

4. 先行信号仪表盘
   ├─ 招聘/薪资趋势
   ├─ 技术社区活跃度
   ├─ 评论速度变化
   └─ 广告/投放信号

5. 战略建议矩阵
   ├─ 高确信度行动项（立即执行）
   ├─ 中确信度假设（需验证）
   └─ 探索性方向（持续监测）
```
